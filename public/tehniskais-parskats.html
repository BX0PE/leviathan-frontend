01 · Kopsavilkums
Kas ir LEVIATHAN

BIS Būvdarbu žurnāla
automatizācijas platforma.

LEVIATHAN ir SaaS lietojumprogramma, kas automatizē datu ievadi Būvniecības informācijas sistēmas (BIS) Būvdarbu žurnālā. Sistēma paredzēta būvfirmu koordinatoriem, priekšniekiem un būvuzraugiem, kuri ikdienā strādā ar tāmēm un tehniskajām deklarācijām.

Vietā, lai ievadītu katru izpildīto darbu manuāli BIS portālā, lietotājs augšupielādē Excel tāmi (LBN 501-17 formāts) un PDF sertifikātus, sistēma automātiski atpazīst pozīcijas, sasaista dokumentus un ļauj priekšniekam laukā mobili ievadīt izpildītos apjomus.

Frontend

Progressive Web App uz React + Vite, ar offline atbalstu (IndexedDB rindā turētie ieraksti, ServiceWorker cache). Reaģē uz navigator.onLine notikumiem, atkārto neveiksmīgas API operācijas, kad savienojums atjaunojas.

Backend

FastAPI (Python 3.12) uz Railway ar PostgreSQL. JWT autentifikācija, BIS OAuth token pārvaldība ar automātisku refresh. Excel parseris (openpyxl) un PDF apstrāde (pdfplumber) tekstu ekstrakcijai un pozīciju sasaistei.

Integrācija

Integrācija izmanto BIS publiski pieejamās REST saskarnes un OAuth 2.0 autentifikāciju. Reģistrētā lietojumprogramma LEVIATHAN testa vidē — statuss Aktīva.

03 · Noturība pret API pārtraukumiem
Ko darām, kad BIS nav pieejams

Integrācija ir uzbūvēta ar
pieņēmumu, ka API var nebūt.

Reālā ekspluatācijā valsts sistēmām nepieciešami apkopes logi, tīkla ierobežojumi vai citas pārtraukuma situācijas. LEVIATHAN ir izstrādāts tā, lai lietotāja darbs neapstātos un dati netiktu zaudēti nevienā scenārijā.

Ziņojums lietotājam pēc "Nosūtīt uz BIS"
"BIS API pagaidām nav pieejams. Ieraksts saglabāts un tiks nosūtīts automātiski, tiklīdz atjaunosies pieeja."

Klienta pusē

IndexedDB rinda — ja lietotājs ievada ierakstu bez interneta, tas glabājas lokāli. Kad navigator.onLine notikums izšaujas, rinda tiek automātiski nosūtīta.

Iepriekš aizpildīti lauki — apraksts un darbinieku skaits tiek prefill'oti no pēdējā ieraksta, lai priekšniekam nav manuāli jāaizpilda katru dienu no jauna.

Servera pusē

PostgreSQL rinda ar synced=False — ja BIS API atbild ar kļūdu, ieraksts saglabājas datubāzē ar karodziņu. Nekas netiek pazaudēts.

Retry endpoint — POST /api/entries/retry-pending atkārto visus nesinhronizētos ierakstus. Izsaukts automātiski, kad savienojums atjaunojas.

Redzamība lietotājam
  • ✓

  • ✓

  • ✓
    Bezsaistes brīdinājums — sticky top-bar informē lietotāju, ka pašlaik nav interneta.
  • ✓
    Audit trail — visas darbības ar objektu, arī neizdevušies BIS mēģinājumi, tiek reģistrētas.
